This package contains Resources implementations for the various supported repositories for static resource files. They may be used (by a Context implementation) in implementing the following API calls:

  • public URL getResource(String path)
  • publid InputStream getResourceAsStream(String path)

The implementations share a common base class that supports caching of resources where appropriate. Caching (and other features) of all the standard Resources implementations are configured by setting the following properties (default values are in square brackets):

  • checkInterval - The interval, in seconds, between checks by our background thread for out-of-date cached resources, or zero for no checks. [0]
  • debug - Debugging detail level that controls our logging, where zero means no debugging messages and higher numbers mean more deatail. [0]
  • docBase - The document base for this Resources implementation, which must follow the syntax specified for each individual implementation. This property is transparently duplicated from the docBase property of the Context we are associated with. [REQUIRED - NO DEFAULT]
  • expand - Should getResource() calls for a "directory" return a directory listing (if "true") or nothing (if "false")? [true]
  • maxCount - The maximum number of resources to cache, or zero for no limit. [0]
  • maxSize - The maximum size (in bytes) of resources to cache, or zero for no limit. [0]
  • minSize - The minimum size (in bytes) of resources to cache, or zero for no limit. [0]

The standard Resources implementations that are currently available include the following (with additional configuration properties as specified):

  • FileResources - Implementation of Resources that operates off a document base that is a directory in the server's local filesystem. The document base must specify a relative or absolute pathname. If it is relative, it is resolved against the application base for our surrounding Host (if any), or against the value of the "catalina.home" system property. Files can be added, removed, or modified within this directory while the application runs -- a background thread periodically checks the last modified time of all cached resources and un-caches entries that have been modified, so that the updated version will be processed the next time it is requested.
  • JarResources - Implementation of Resources that operates off a web application archive (WAR) file directly. In this environment, there is no concept of "file paths" to the various entries, so getRealPath() will always return null. In addition, this implementation assumes that the individual entries in the WAR file, or the WAR file itself, will not be modified while the application is running -- therefore, no checking for out-of-date cache entries is required. Caching is still performed, subject to the policies enforced by the cacheable() method, because the JAR file itself might be on a remote server.
